Requisitos de Sistema para Zabbix e Nagios

Esta página fornece um resumo dos requisitos de sistema para as ferramentas de monitoramento Zabbix e Nagios. Use estas informações para planejar sua instalação e garantir que seu ambiente atenda às necessidades de cada ferramenta.

Zabbix

Zabbix é uma solução de monitoramento de nível empresarial de código aberto para monitorar a disponibilidade e o desempenho de aplicações, serviços, servidores e hardware de rede.

Requisitos de Hardware

  • Servidor: Mínimo de 2 cores de CPU, 2GB de RAM, 20GB de espaço em disco
  • Proxy: Mínimo de 1 core de CPU, 512MB de RAM, 10GB de espaço em disco
  • Agente: Requisitos mínimos, depende dos itens monitorados

Os requisitos de hardware podem variar dependendo do número de dispositivos e métricas monitoradas. Para ambientes maiores, mais recursos de CPU, RAM e disco são recomendados.

Requisitos de Software

  • Sistema Operacional: Linux (CentOS, Debian, Ubuntu), Windows Server
  • Banco de Dados: MySQL, PostgreSQL, Oracle, SQLite, IBM DB2
  • Servidor Web: Apache, Nginx
  • PHP: Versão 7.2 ou superior (para a interface web)

Zabbix Server requer um banco de dados para armazenar dados de configuração e monitoramento. A interface web é opcional, mas recomendada para configuração e visualização de dados.

Nagios

Nagios é um sistema de monitoramento de rede amplamente utilizado para monitorar serviços, sistemas e processos. Ele alerta os usuários sobre condições críticas e erros, ajudando-os a minimizar interrupções e perdas.

Requisitos de Hardware

  • Servidor: Mínimo de 1GHz CPU, 512MB RAM, 40GB de espaço em disco
  • Agente: Requisitos mínimos, depende dos checks monitorados

Os requisitos de hardware do Nagios podem aumentar com o número de hosts e serviços monitorados, a frequência dos checks e os métodos de notificação configurados.

Requisitos de Software

  • Sistema Operacional: Linux (RHEL, CentOS, Ubuntu), Unix
  • Servidor Web: Apache
  • PHP: Necessário para a interface web
  • Compilador C: Necessário para compilar plugins

Nagios Core é o mecanismo de monitoramento, e a interface web (opcional) fornece uma interface para visualização do status do monitoramento, configuração e relatórios. Plugins são necessários para monitorar serviços específicos.
